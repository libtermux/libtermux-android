Installation GETTING STARTED

Add to your project

Add JitPack to your root settings.gradle.kts:

Kotlin DSL 
dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        maven { url = uri("https://jitpack.io") }
    }
}

Module dependencies

Add only what your app needs:

build.gradle.kts 
dependencies {
    // Core — always required (bootstrap, CLI, bridge)
    implementation("com.github.AeonCoreX-Lab:libtermux-android:1.0.0")

    // OS Module — add only if you need Linux distro support
    implementation("com.github.AeonCoreX-Lab:libtermux-os:1.0.0")

    // Terminal UI widget — optional
    implementation("com.github.AeonCoreX-Lab:libtermux-terminal-view:1.0.0")
}

    class MyApp : Application() {

            autoInstall        = true
            logLevel           = LogLevel.DEBUG
            maxCommandTimeoutMs = 60_000L
            env("MY_VAR", "hello")
        })
    }
}
  2. 2

    Install the bootstrap (once)

    Downloads and extracts Termux binaries on first run (~40 MB).

    Kotlin (ViewModel) 
    viewModelScope.launch {
    app.libtermux.install().collect { state ->
        when (state) {
            is InstallState.Downloading  -> updateProgress(state.progress)
            is InstallState.Extracting   -> showExtracting()
            is InstallState.Completed    -> onReady()
            is InstallState.Failed       -> showError(state.reason)
            is InstallState.AlreadyInstalled -> onReady()
            else -> {}
        }
    }
}

Configuration DSL

Kotlin 
val config = termuxConfig {
    // Target CPU architecture (AUTO detects at runtime)
    architecture = Architecture.AUTO  // ARM64 | X86_64 | ARM | X86

    // Auto-install bootstrap on first launch
    autoInstall = true

    // Pin a specific bootstrap version (default: latest)
    bootstrapVersion = TermuxConfig.LATEST_BOOTSTRAP

    // Logging level
    logLevel = LogLevel.INFO  // VERBOSE | DEBUG | INFO | WARN | ERROR | NONE

    // Max execution time per command
    maxCommandTimeoutMs = 30_000L

    // Inject extra environment variables
    env("CUSTOM_PATH", "/data/data/com.example/files/bin")
    env("API_KEY", BuildConfig.TERMUX_KEY)

    // Custom bootstrap download URL (optional)
    customBootstrapUrl = "https://your-cdn.com/bootstrap-arm64.zip"
}

Java users can use TermuxConfig.builder().architecture(Architecture.ARM64).build() instead of the Kotlin DSL.

TermuxBridge

The primary interface for running code in the Termux environment. All methods are suspend.

Kotlin 
val bridge = libtermux.bridge

// Shell commands
bridge.run("ls -la /etc")
bridge.runOrThrow("git status")  // throws on non-zero exit
bridge.runStreaming("tail -f /var/log/syslog").collect { line -> log(line) }

// Language runners
bridge.bash("for i in 1 2 3; do echo \$i; done")
bridge.python("import sys; print(sys.version)", args = listOf("--verbose"))
bridge.node("console.log(process.version)")
bridge.ruby("puts RUBY_VERSION")
bridge.perl("print \$^V")
bridge.php("echo PHP_VERSION;")

// File operations
bridge.writeFile("hello.py", "print('hello world')")
bridge.readFile("hello.py")
bridge.fileExists("hello.py")

// Package management
bridge.install("nmap", "curl", "git")
bridge.uninstall("nano")
bridge.upgradeAll()
bridge.pipInstall("requests", "numpy")
bridge.npmInstall("express")

// System info
bridge.hasCommand("python3")
bridge.getKernelInfo()
bridge.getEnvironment()

CommandExecutor

Low-level executor used by TermuxBridge internally. Use directly when you need full control over process execution.

EACCES Auto-Recovery

If ProcessBuilder throws Permission denied (error=13), the executor automatically calls Os.chmod() on the shell binary and retries once — self-healing without requiring reinstall.

val result: ExecutionResult = executor.execute(
    command  = "nmap -sV 192.168.1.1",
    workDir  = File(vfs.homeDir, "scans"),
    extraEnv = mapOf("TERM" to "xterm-256color"),
    shell    = "bash",         // which shell binary to use
)

println(result.stdout)         // command output
println(result.stderr)         // error output
println(result.exitCode)       // 0 = success
println(result.executionTimeMs)
println(result.isSuccess)      // exitCode == 0

OS Module Overview :OS MODULE

The :os module adds full Linux distro support — download, extract, and run Kali, Ubuntu, Debian, Alpine, or Fedora rootfs environments with a full graphical desktop inside Jetpack Compose.

No Circular Dependency

:core has zero knowledge of :os. The bridge is the LibTermux.createOs() extension function defined in :os.

Kotlin 
// Create OS environment (extension from :os module)
val os = libtermux.createOs {
    executionMode = ExecutionMode.AUTO  // auto-detect root

    registry {
        distro(Distro.Kali) {
            guiEnabled         = true
            desktopEnvironment = DesktopEnvironment.GNOME_FLASHBACK
            defaultResolution  = DisplayResolution.FHD_1080P
            description        = "Penetration testing suite"
            extraPackages      = listOf("nmap", "metasploit-framework")
            startupCommands    = listOf("service postgresql start")
        }
        distro(Distro.Ubuntu2404) {
            guiEnabled    = false
            extraPackages = listOf("python3-pip", "nodejs", "git")
        }
    }
}

// Install a distro
os.setupDistro(Distro.Kali).collect { state ->
    when (state) {
        is DistroSetupState.Downloading -> updateBar(state.progress)
        is DistroSetupState.Completed   -> onReady()
        is DistroSetupState.Failed      -> showError(state.reason)
        else -> {}
    }
}

// Run CLI commands
val result = os.execute(Distro.Kali, "nmap -sV 192.168.1.1")

// Install packages
os.install(Distro.Ubuntu2404, "htop", "tree", "ripgrep")

Distro Registry

Declare exactly which distros your app supports. Only registered distros appear in DistroLauncher — preventing accidental 2 GB rootfs downloads.

registry {
    distro(Distro.Kali) {
        guiEnabled         = true
        desktopEnvironment = DesktopEnvironment.XFCE4  // or GNOME_FLASHBACK
        defaultResolution  = DisplayResolution.HD_720P
        colorDepth         = 24
        vncDisplay         = 1           // port = 5900 + display = 5901
        vncPassword        = null        // null = no auth (localhost only)
        startupCommands    = listOf(     // run each session start
            "service postgresql start",
            "service ssh start",
        )
        extraPackages      = listOf("nmap", "wireshark-common")
        extraBindMounts    = listOf("/sdcard/hacking:/hacking")
        showInLauncher     = true
        description        = "Kali Linux penetration testing"
    }
}

proot vs Real chroot

LibTermux automatically selects the best backend via ExecutionMode.AUTO. Override with ExecutionMode.PROOT or ExecutionMode.REAL_CHROOT.

proot (No Root)

  • Any Android device
  • App Store safe
  • apt / apk / dnf work
  • GUI desktop via VNC
  • ~85% native speed
  • Raw sockets
  • iptables / nftables
  • systemd (not supported)
  • Kernel modules

Real chroot (Root)

  • 100% native speed
  • Raw sockets
  • iptables / nftables
  • Network capture (tcpdump)
  • Full bind mounts
  • Preserved setuid bits
  • systemd (partial)
  • Kernel modules
  • Play Store (rooted only)

GNOME on Android

Full GNOME Shell requires Mesa LLVMpipe software rendering (LIBGL_ALWAYS_SOFTWARE=1) since proot has no GPU passthrough. This is handled automatically. Use GNOME Flashback for best performance — it uses Metacity WM instead of gnome-shell, reducing RAM usage by ~600 MB.

GUI Desktop Session

Kotlin 
// 1. Create session
val session = os.createDesktopSession(Distro.Kali)

// 2. Start with settings
session.start(DistroRuntimeSettings(
    distroId      = Distro.Kali.id,
    displayWidth  = 1920,
    displayHeight = 1080,
    colorDepth    = 24,
    vncPort       = 5901,
    scaleToFit    = true,
    showToolbar   = true,
))

// 3. Observe state
session.sessionState.collect { state ->
    when (state) {
        is DesktopSessionState.InstallingPackage -> showMsg("Installing: ${state.pkg}")
        is DesktopSessionState.WaitingForServer  -> showMsg("Waiting ${state.attempt}s...")
        is DesktopSessionState.Running           -> hideLoading()
        is DesktopSessionState.Failed            -> showError(state.reason)
        else -> {}
    }
}

// 4. Stop when done
session.stop()

DistroDisplay Composable

Renders the VNC framebuffer in a Jetpack Compose Canvas with full touch and keyboard input.

@Composable
fun DesktopScreen(session: DesktopSession) {
    DistroDisplay(
        session     = session,
        modifier    = Modifier.fillMaxSize(),
        showToolbar = true,           // overlay toolbar with Ctrl+Alt+Del etc.
        onClose     = { /* navigate back */ },
    )
}

// Or use the full launcher UI
@Composable
fun DistrosScreen(os: OsEnvironment) {
    DistroLauncher(
        os           = os,
        onLaunch     = { session -> navTo("desktop") },
        onOpenSettings = { distro -> navTo("settings/${distro.id}") },
    )
}

Distro Settings

Per-distro settings are persisted via DataStore Preferences and survive app restarts.

val store = DistroSettingsStore(context)

// Observe
store.getSettings(Distro.Kali).collect { settings ->
    println("${settings.displayWidth}×${settings.displayHeight}")
}

// Update
store.update(Distro.Kali) { it.copy(displayWidth = 1920, displayHeight = 1080) }

// Reset to defaults
store.reset(Distro.Kali)

FAQ

Does this work without root?

Yes. The :core bootstrap and all :os distro environments work without root using proot. Root unlocks native-speed chroot and raw network access.

Can I use this for a Play Store app?

For specific-purpose apps (Python IDE, Git client, SSH tool) — yes. General-purpose terminals and package managers are risky. Penetration testing / Kali tools will be rejected. Use F-Droid / GitHub release for those.

How large is the bootstrap?

The Termux bootstrap (~40 MB compressed, ~120 MB extracted) is downloaded on first run. Distro rootfs sizes range from 5 MB (Alpine) to 400 MB (Kali). All stored in app private storage.

What Android version is required?

Minimum Android 8.0 (API 26). The OS module with Compose requires Android 8.0+ as well.

Can I run a full GNOME desktop?

Yes. GNOME Shell runs via X11 with Mesa software rendering. Use DesktopEnvironment.GNOME_FLASHBACK for best performance — it uses Metacity WM and runs on ~450 MB instead of ~1.5 GB.
